# 修复 Claude 桌面端安装 "Download failed" 错误

## 问题描述

安装 Claude Desktop 时弹出错误：

> Download failed. Check your internet connection and try again.
>
> Diagnostic log saved to: C:\Users\<用户名>\AppData\Local\Temp\ClaudeSetup.log

## 原因分析

Claude Desktop 安装程序需要在安装过程中从 Anthropic 服务器下载额外文件。此错误通常由以下原因导致：

1. **网络连接问题** — 无法访问 Anthropic 的下载服务器
2. **防火墙/代理拦截** — 企业网络或安全软件阻止了下载请求
3. **DNS 解析失败** — 无法解析下载域名
4. **网络不稳定** — 下载过程中连接中断

## 解决方案

### 方案一：使用 VPN 或代理（最常见有效）

Claude 的下载服务器在海外，国内网络可能无法直接访问。

1. 开启可用的 VPN 或网络代理工具
2. 确认代理已开启「全局模式」（而非仅浏览器代理）
3. 重新运行 Claude 安装程序

### 方案二：配置系统代理

如果你有 HTTP 代理，可以在安装前配置系统环境变量：

```powershell
# 在 PowerShell 中设置（替换为你的代理地址和端口）
$env:HTTP_PROXY = "http://127.0.0.1:7890"
$env:HTTPS_PROXY = "http://127.0.0.1:7890"

# 然后运行安装程序
& "C:\路径\Claude-Setup.exe"
```

### 方案三：手动下载离线安装包

如果在线安装始终失败，可以尝试通过其他方式获取完整安装包：

1. 前往 [Claude 官方下载页面](https://claude.ai/download)
2. 在网络通畅的环境下下载完整安装包
3. 将安装包拷贝到目标电脑上安装

### 方案四：检查防火墙和安全软件

1. 临时关闭 Windows Defender 防火墙：
   - 打开「Windows 安全中心」→「防火墙和网络保护」
   - 临时关闭当前活动网络的防火墙
2. 如果安装了第三方杀毒软件（如 360、火绒等），临时退出或添加安装程序为信任
3. 安装完成后记得重新开启防火墙

### 方案五：更换 DNS

将 DNS 服务器改为更可靠的公共 DNS：

1. 打开「控制面板」→「网络和共享中心」→「更改适配器设置」
2. 右键当前网络连接 →「属性」
3. 双击「Internet 协议版本 4 (TCP/IPv4)」
4. 选择「使用下面的 DNS 服务器地址」
5. 设置为：
   - 首选 DNS：`8.8.8.8`
   - 备用 DNS：`8.8.4.4`
6. 点击确定后重新尝试安装

### 方案六：查看诊断日志

如果以上方案均不奏效，可以查看安装日志获取更多信息：

1. 按 `Win + R`，输入 `%TEMP%` 打开临时文件夹
2. 找到 `ClaudeSetup.log` 文件
3. 用记事本打开，查看具体的错误信息
4. 日志中通常会显示具体是哪个 URL 下载失败，有助于进一步排查

## 常见补充提示

- 安装包要以**管理员身份运行**（右键 → 以管理员身份运行）
- 确保 Windows 系统时间正确（时间偏差可能导致 HTTPS 证书验证失败）
- 如果使用公司网络，咨询 IT 部门是否对外部下载有限制
