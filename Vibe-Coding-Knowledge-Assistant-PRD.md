# Vibe Coding 行业知识库助手 - PRD

## 1. 产品概述

一款面向 Vibe Coding 从业者、创业者和技术爱好者的行业知识库助手。产品自动聚合来自 X（Twitter）、Product Hunt、行业博客等多渠道的 Vibe Coding 热点资讯，同时支持用户自主收藏和上传资料。所有内容在保留原始源文件的基础上，由 AI 自动生成结构化摘要、标签、核心观点提炼、反驳视角和金句集锦，帮助用户高效追踪行业动态、沉淀知识体系、培养批判性思维。

## 2. 目标用户

| 用户类型 | 描述 |
|----------|------|
| Vibe Coding 开发者 | 使用 Cursor、Claude Code、Replit 等工具的开发者，需要追踪最新工具动态和最佳实践 |
| 技术创业者 / 独立开发者 | 利用 Vibe Coding 工具快速构建产品的创始人，关注行业趋势和竞品动态 |
| 产品经理 / 技术决策者 | 需要评估 Vibe Coding 工具和方法论，为团队做技术选型决策 |
| 技术内容创作者 | 撰写 Vibe Coding 相关内容的博主、自媒体作者，需要系统化素材管理 |

## 3. 核心功能模块

### 3.1 热点资讯聚合模块

**功能描述：** 自动抓取和聚合多渠道的 Vibe Coding 行业资讯，为用户呈现最新热点动态。

| 功能项 | 说明 |
|--------|------|
| X（Twitter）热门案例推荐 | 追踪 Vibe Coding 相关 KOL（如 @kaborosky、@levelsio、@mckaywrigley 等）和热门话题标签（#vibecoding、#vibeShipping、#buildInPublic），自动筛选高互动量推文（点赞 > 100 / 转发 > 50），展示热门 Vibe Coding 案例 |
| Product Hunt 热门产品追踪 | 每日自动抓取 Product Hunt 上与 Vibe Coding / AI Coding 相关的新产品及其排名、投票数、用户评论摘要，支持按日/周/月维度查看趋势 |
| 行业新闻聚合 | 聚合来自 TechCrunch、The New Stack、InfoQ、Hacker News、Dev.to 等技术媒体的 Vibe Coding 相关文章，按热度和时效性排序 |
| 热点时间线 | 按时间线展示重大行业事件（如新工具发布、融资消息、重要技术突破），形成可回溯的行业大事记 |
| 智能推荐 | 基于用户阅读历史和收藏偏好，个性化推荐相关内容 |

### 3.2 个人知识库模块

**功能描述：** 支持用户主动收集、整理和管理自己感兴趣的 Vibe Coding 相关资料，构建个人知识体系。

| 功能项 | 说明 |
|--------|------|
| 网页收藏 | 支持通过 URL 链接收藏任意网页（官网博客、技术文章、教程等），自动提取正文内容并保留原始链接 |
| 文件上传 | 支持上传行业文件（PDF / Word / Markdown / TXT / EPUB），系统自动解析文件内容 |
| 浏览器插件（扩展） | 提供 Chrome / Edge 插件，一键将当前网页保存到知识库 |
| 源文件保留 | 所有收藏和上传的内容均保留原始格式和来源链接，确保可溯源 |
| 文件夹 / 分类管理 | 支持用户自建文件夹，按主题或项目对资料进行分类管理 |
| 全文搜索 | 支持对所有已收录内容进行全文检索，快速定位目标资料 |

### 3.3 AI 智能分析模块

**功能描述：** 对所有收录的内容（无论来自自动聚合还是用户上传）自动进行 AI 深度分析，生成结构化知识卡片。

每篇内容将自动生成以下结构化分析结果：

#### 3.3.1 自动标签生成

| 功能项 | 说明 |
|--------|------|
| 主题标签 | 自动识别内容所属领域（如 `#AI编程`、`#Cursor`、`#代码安全`、`#独立开发`） |
| 情感标签 | 标注内容的立场和基调（如 `看好`、`质疑`、`中立分析`、`教程`） |
| 工具标签 | 自动提取文中涉及的工具和产品名称（如 `Cursor`、`Replit`、`Bolt`、`v0`） |

#### 3.3.2 文章摘要

用 3-5 句话概括全文的核心内容、论证逻辑和最终结论。要求信息密度高，读完摘要就能把握文章全貌。

**示例输出：**

> 本文分析了 2026 年 Vibe Coding 从"生成代码"向"交付产品"（Vibe Shipping）的范式转移，指出以 Base44、Lovable 为代表的全栈平台正在取代传统开发流程。作者通过对比 10 款主流工具的性能数据，论证了非技术创始人已具备独立交付 SaaS 产品的能力。文章同时警告，AI 生成代码的安全漏洞率（24.7%）和"认知债务"问题将成为行业最大隐患，建议采用人机协同的审查机制。

#### 3.3.3 核心观点

逐条提炼文章的主要论点，每条包含一个加粗的小标题和 1-2 句解释。忠实于原文，不添加作者未表达的观点。

**示例输出：**

- **Vibe Shipping 是 Vibe Coding 的必然进化** — 创始人需要的不是代码文件，而是可直接分享给客户的上线产品，全栈 AI 平台正在填补从原型到部署的最后一公里。
- **安全债务正在取代技术债务** — AI 生成代码的速度远超人工审查能力，24.7% 的安全缺陷率意味着每四行 AI 代码就有一行存在潜在风险。
- **经验差距决定生产力收益** — 资深开发者（10 年以上）获得 81% 的效率提升，而初级开发者中有 40% 承认在不完全理解的情况下部署了 AI 生成的代码。

#### 3.3.4 AI 反驳观点

从不同角度对文章核心论点提出质疑，辅助用户进行批判性思考。要求：
- 每条反驳针对一个具体论点，不泛泛而谈
- 指出论证中的假设、盲区或逻辑跳跃
- 语气理性客观，不是为了反驳而反驳

**示例输出：**

- **"Vibe Shipping"可能混淆了"部署"与"交付"** — 文章将"一键部署"等同于"产品交付"，但真正的产品交付还需要用户研究、运维监控、持续迭代等环节，仅靠 AI 生成代码无法覆盖完整的产品生命周期。
- **24.7% 安全缺陷率缺少对照基准** — 文章未给出同期人工编写代码的安全缺陷率作为对比，无法判断这个数字是 AI 特有的问题还是行业普遍水平。如果人工代码同样存在高缺陷率，那么问题的根源可能是安全审查流程而非 AI 本身。
- **经验差距的论述存在选择偏差** — 资深开发者的高生产力收益可能源于他们本身更善于分解任务和评估代码质量，而非 Vibe Coding 工具的直接功劳。初级开发者的问题也许更多是教育体系的缺失，而非工具的固有缺陷。

#### 3.3.5 金句集锦

提取原文中最精彩、最有传播力的句子（3-6 句），用引用格式呈现。优先选择：观点鲜明的判断句、精妙的比喻、高度浓缩的洞察。

**示例输出：**

> "Founders don't want code files — they want live, deployed applications they can share with customers."

> "Cognitive debt is the new technical debt."

> "Vibe coding is great for prototypes but not wonderful for brownfield or production code."

> "When open-source projects depend on user engagement for returns, widespread vibe coding creates a negative feedback loop."

### 3.4 知识发现与洞察模块

**功能描述：** 在海量知识库基础上，提供趋势分析和深度洞察能力。

| 功能项 | 说明 |
|--------|------|
| 趋势看板 | 按周 / 月统计热门标签、工具提及频次、话题热度变化曲线 |
| 竞品对比 | 选择多款 Vibe Coding 工具，自动从知识库中聚合相关评测和用户反馈，生成对比报告 |
| 知识图谱 | 可视化展示工具、概念、人物之间的关联关系（如 Cursor → AI IDE → Anysphere → $900M 融资） |
| AI 问答 | 基于知识库内容进行对话式问答（如"Cursor 和 Windsurf 的主要区别是什么？""2026 年哪些 Vibe Coding 工具获得了融资？"） |
| 周报 / 月报自动生成 | 根据设定的关注主题，自动生成行业周报 / 月报，涵盖关键事件、热门观点和数据变化 |

## 5. AI 分析流程设计

```
内容输入
  │
  ├── 自动聚合（X / Product Hunt / 新闻源）
  │
  └── 用户主动（URL 收藏 / 文件上传 / 插件保存）
  │
  ▼
内容解析层
  │
  ├── 网页正文提取（去除广告、导航等噪音）
  ├── 文件格式解析（PDF / Word / Markdown）
  └── 多语言检测与处理
  │
  ▼
AI 分析层（调用 LLM）
  │
  ├── 自动标签生成（主题 / 情感 / 工具标签）
  ├── 文章摘要（3-5 句核心概括）
  ├── 核心观点提炼（逐条加粗标题 + 解释）
  ├── AI 反驳观点（针对性质疑 + 盲区指出）
  └── 金句集锦（3-6 句精华提取）
  │
  ▼
结构化存储
  │
  ├── 原始内容（全文保留）
  ├── AI 分析结果（JSON 结构化存储）
  ├── 标签索引
  └── 向量嵌入（用于语义搜索和关联推荐）
```

## 6. 技术建议

| 层级 | 推荐方案 |
|------|---------|
| 前端 | Next.js + TailwindCSS + shadcn/ui，响应式设计兼容移动端 |
| 后端 | Node.js（Express / Fastify）或 Python（FastAPI） |
| 数据库 | PostgreSQL（结构化数据） + Pinecone / Weaviate（向量数据库，用于语义搜索） |
| AI 模型 | Claude API（文章分析、摘要生成、观点提炼）；嵌入模型用于向量化 |
| 数据抓取 | X API v2（推文抓取）；Product Hunt GraphQL API；RSS + 爬虫（新闻聚合） |
| 内容解析 | Mozilla Readability（网页正文提取）；pdf-parse / mammoth（文件解析） |
| 搜索引擎 | Elasticsearch / Meilisearch（全文检索） |
| 浏览器插件 | Chrome Extension Manifest V3 |
| 部署 | Vercel（前端）+ Railway / Fly.io（后端）+ Supabase（数据库托管） |
| 定时任务 | Cron Jobs / BullMQ（内容定时抓取与分析队列） |

## 7. 数据模型设计（核心）

```
Article（文章/内容）
├── id: UUID
├── title: String              // 标题
├── source_type: Enum          // 来源类型（x_tweet / product_hunt / news / user_url / user_upload）
├── source_url: String         // 原始链接
├── raw_content: Text          // 原始内容全文
├── file_path: String          // 上传文件存储路径（如有）
├── language: String           // 语言
├── created_at: DateTime
├── updated_at: DateTime
│
├── AI Analysis（AI 分析结果）
│   ├── summary: Text              // 文章摘要（3-5 句）
│   ├── core_viewpoints: JSON      // 核心观点列表
│   ├── counter_arguments: JSON    // AI 反驳观点列表
│   ├── golden_quotes: JSON        // 金句集锦列表
│   └── analysis_status: Enum      // 分析状态（pending / processing / completed / failed）
│
├── Tags（标签）
│   ├── auto_tags: String[]        // AI 自动生成标签
│   └── manual_tags: String[]      // 用户手动添加标签
│
└── Metadata（元数据）
    ├── author: String
    ├── publish_date: DateTime
    ├── engagement: JSON           // 互动数据（点赞、转发、评论数等）
    └── product_hunt_rank: Integer // PH 排名（如适用）
```

## 8. MVP 范围（第一版）

优先实现以下功能，后续迭代扩展：

1. **热点资讯浏览** — X 热门推文展示 + Product Hunt 每日精选 + 行业新闻聚合，按热度排序
2. **个人知识库** — 支持 URL 收藏和文件上传（PDF / Markdown / TXT），保留原始内容
3. **AI 自动分析** — 对所有收录内容自动生成标签、摘要、核心观点、反驳观点、金句集锦五项结构化分析
4. **全文搜索** — 支持按关键词和标签检索所有已收录内容
5. **内容详情页** — 左侧原文展示 + 右侧 AI 分析卡片的双栏布局

**MVP 不包含（后续版本迭代）：**
- 浏览器插件
- 知识图谱可视化
- AI 对话问答
- 自动生成周报/月报
- 竞品对比报告

## 9. 后续扩展方向

- **浏览器插件** — Chrome / Edge 一键收藏插件，无需离开当前页面
- **AI 对话问答** — 基于知识库的 RAG 问答系统，支持自然语言检索
- **知识图谱** — 可视化工具、概念、人物之间的关联网络
- **自动周报 / 月报** — 根据关注主题定期生成行业摘要报告
- **团队协作** — 支持团队共享知识库、协同标注和讨论
- **移动端 App** — iOS / Android 原生应用，支持离线阅读
- **Webhook / API 集成** — 对接 Notion、飞书、Slack 等工具，实现跨平台知识同步
- **多语言自动翻译** — 非中文内容自动翻译，降低阅读门槛
- **RSS 自定义订阅** — 用户自行添加 RSS 源，扩展内容来源
